🎯 HOW TO USE
================================================================================

1️⃣  START EVERYTHING
    ┌────────────────────────────────────────┐
    │  bash setup.sh                         │
    └────────────────────────────────────────┘
    
    This single command:
    ✓ Generates TLS certificates
    ✓ Starts 3-node Elasticsearch cluster
    ✓ Configures security and passwords
    ✓ Sets up ILM policies
    ✓ Starts Kibana with SIEM
    ✓ Deploys Fleet Server
    ✓ Starts Logstash
    ✓ Starts Customer Service app
    
    ⏱️  Takes 5-10 minutes
    

2️⃣  ACCESS SERVICES
    ┌────────────────────────────────────────┐
    │  Kibana:        http://localhost:5601  │
    │  Elasticsearch: https://localhost:9200 │
    │  Customer API:  http://localhost:8081  │
    └────────────────────────────────────────┘
    
    Login: elastic / SecurePassword123!


3️⃣  CHECK STATUS
    ┌────────────────────────────────────────┐
    │  bash status.sh                        │
    └────────────────────────────────────────┘
    
    Shows:
    • Docker containers status
    • Elasticsearch cluster health
    • Kibana accessibility
    • Customer service status


4️⃣  GENERATE TEST EVENTS
    ┌────────────────────────────────────────┐
    │  bash simulate-events.sh               │
    └────────────────────────────────────────┘
    
    Creates:
    • SSH brute force attempts
    • Failed login attempts
    • Suspicious network connections
    • Data exfiltration events
    • Webshell activity
    
    View alerts: Kibana → Security → Alerts


5️⃣  STOP EVERYTHING
    ┌────────────────────────────────────────┐
    │  bash stop.sh                          │
    └────────────────────────────────────────┘
    
    Safely stops all services
    (Data is preserved)

================================================================================
🎓 DETECTION RULES CONFIGURED
================================================================================

1. SSH Brute Force Activity
   → 5+ failed SSH attempts from same IP in 5 min
   → Severity: High

2. Multiple Failed Login Attempts
   → 10+ authentication failures in 5 min
   → Severity: High

3. Suspicious Network Connection
   → Connections to C2 ports (4444, 5555, etc.)
   → Severity: Critical

4. Potential Data Exfiltration
   → 100+ outbound connections in 15 min
   → Severity: High

5. Webshell Activity
   → Web requests with shell command patterns
   → Severity: Critical

================================================================================
🔧 CUSTOMIZATION
================================================================================

Edit .env file to change:
  • Passwords (ELASTIC_PASSWORD, KIBANA_PASSWORD)
  • Ports (ES_PORT, KB_PORT, LS_PORT)
  • Memory limits (ES_JAVA_OPTS, LS_JAVA_OPTS)
  • Cluster name (CLUSTER_NAME)

After changes: bash stop.sh && bash setup.sh

📖 ADDITIONAL COMMANDS
================================================================================

View logs:
  sudo docker-compose logs -f [service-name]
  
Examples:
  sudo docker-compose logs -f es01
  sudo docker-compose logs -f kibana
  sudo docker-compose logs -f logstash

Cluster health:
  curl -k -u elastic:SecurePassword123! \
    https://localhost:9200/_cluster/health?pretty

List indices:
  curl -k -u elastic:SecurePassword123! \
    https://localhost:9200/_cat/indices?v

Restart service:
  sudo docker-compose restart [service-name]
